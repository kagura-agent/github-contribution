### The openclaw Post-Mortem

This deserves its own section because it's the worst execution failure in 5 weeks:

- **Week 3 (May 17):** Identified that openclaw requires behavioral proofs. Set action item to study.
- **Week 4 (May 24):** #83378 received `proof: sufficient` + `waiting on author`. Noted as "priority 1 for next week."
- **Week 5 (May 31):** Never responded. PR closed. All 5 PRs closed in batch.

**What happened:** The cron job kept searching for new repos and submitting new PRs instead of doing the _maintenance_ work on existing PRs. The action item system has no enforcement mechanism — writing "priority 1" changes nothing if the execution loop doesn't check it.

**Lesson:** Carryover PR maintenance (responding to reviews, updating branches) must be a separate tracked activity, not just a note in the reflection. If `waiting on author` exists, it should trigger a specific cron/reminder — not rely on a human-style "I'll remember."
